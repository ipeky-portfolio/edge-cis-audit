# Microsoft Edge Security Audit – CIS-ähnliche Benchmarks

## Übersicht

Dieses Projekt zeigt ein Security-Audit für Microsoft Edge.  
Die Bewertung basiert auf Best Practices für Browser-Härtung, wie sie in CIS Benchmarks für Webbrowser empfohlen werden.

## Ziele

- Überprüfung sicherer Edge-Konfigurationen
- Identifikation von Abweichungen
- Risikobasierte Bewertung
- Handlungsempfehlungen

## Scope

- Microsoft Edge Version ≥ Stable / Chromium-basiert
- Windows Clients 10/11
- User-Profile
- Enterprise Policies (MDM / GPO)

## Audit-Kategorien

- Safe Browsing / Phishing-Schutz
- HTTPS / SSL Enforcement
- Passwort & Autofill
- Erweiterungen / Add-ons
- Telemetrie / Datenfreigabe
- Pop-ups / Downloads / Plugins
- Proxy / Netzwerk-Einstellungen
- Update Policy

## Bewertungskriterien

- Konform
- Nicht konform
- Nicht anwendbar
- Manuelle Prüfung erforderlich

## Zielgruppe

IT-Security Teams, Auditoren, DevSecOps
